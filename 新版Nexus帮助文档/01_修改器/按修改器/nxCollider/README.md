# nx 碰撞体（nxCollider）

当前文档版本：NeXus `2026.0.7011503`

`nx 碰撞体（nxCollider）` 负责把指定对象变成粒子要撞上的表面。  
当前版本里，它可以接收：

- 网格（Mesh）对象
- 曲线（Curves）对象

它的职责很集中：

- 决定粒子会不会撞到这些对象
- 决定从哪一侧开始碰撞
- 决定碰撞后的反弹、滑动和散开程度

## 当前界面怎么理解

当前版本里，`nx 碰撞体（nxCollider）` 的核心非常集中，重点就是碰撞体（Colliders）列表和每个列表项的碰撞设置。

最关键的内容只有这几项：

- 碰撞体（Colliders）
- 法线方向（Normals）
- 弹力（Bounce）
- 摩擦力（Friction）
- 散射（Scatter）

用户可以很直接地把“加一个碰撞物体”和“粒子行为发生变化”对应起来。

## 一条最容易跑通的起步方法

1. 先确认 `nx 发射器（nxEmitter）` 已经能稳定产生粒子。
2. 先确认 `nx 缓存（nxCache）` 已经把上游结果固定下来。
3. 创建 `nx 碰撞体（nxCollider）`。
4. 在碰撞体（Colliders）列表里加入一个最容易观察的对象，例如地面。
5. 先用外部（Outside）测试。
6. 再根据反弹和滑动的结果调整弹力（Bounce）与摩擦力（Friction）。

这样做的优点是：  
你先只解决“会不会撞上去”，然后再解决“撞上去之后像什么”。

## 先抓住这几个最关键的控制项

### 碰撞体（Colliders）

碰撞体（Colliders）列表决定哪些对象参与碰撞。

当前版本里，每一项都可以单独启用或停用，也有自己的一套碰撞参数。  
如果你想让粒子撞多个对象，就在这里加多个条目。

建议先只加一个最容易观察的对象，例如：

- 地面
- 一面墙
- 一个简单容器

这样更容易判断碰撞有没有先跑通。

### 法线方向（Normals）

法线方向（Normals）决定粒子从哪一侧和对象发生碰撞。当前版本里有：

- 外部（Outside）
- 内部（Inside）
- 任意（Any）

可以这样理解：

- 外部（Outside）：更适合普通外表面
- 内部（Inside）：更适合容器内壁这类场景
- 任意（Any）：两侧都参与碰撞，适合排查法线方向问题

如果你明明看到粒子穿过对象，优先先看这里。  
排查时最常用的办法，是先临时切到任意（Any），确认到底是碰撞没生效，还是法线侧面选错了。

### 弹力（Bounce）

弹力（Bounce）控制粒子碰撞后还保留多少反弹趋势。

- 值越高，越容易弹开
- 值越低，越像把速度吃掉一部分

想要颗粒、石子、硬表面反弹感更强时，先提高弹力（Bounce）。  
想让结果更沉、更稳、更不容易跳时，先降低弹力（Bounce）。

### 摩擦力（Friction）

摩擦力（Friction）控制粒子碰撞后沿表面滑动时的阻力。

- 值越高，越不容易滑走
- 值越低，越容易顺着表面继续滑

如果粒子落到斜面后一直滑得太快，先提高摩擦力（Friction）。  
如果你希望粒子更顺着表面流动，再把它放低。

### 散射（Scatter）

散射（Scatter）控制碰撞后方向被打散的程度。

- 值越高，碰撞后的方向越乱
- 值越低，碰撞后的轨迹越可预测

如果你想让粒子撞上粗糙表面后四散开，先提高散射（Scatter）。  
如果你只是想做稳定可控的基础碰撞，通常先把散射（Scatter）保持低一些。

## 它处理的碰撞范围要怎么理解

`nx 碰撞体（nxCollider）` 当前这篇基础文档里，最重要的一条认识是：

它处理的是粒子和场景对象之间的碰撞。

如果你需要粒子和粒子之间互相碰撞，当前要关注的是 `nx 粒子-粒子碰撞（nxPPCollisions）`，不是 `nx 碰撞体（nxCollider）`。

这两个方向都常用，但解决的问题不同。

## 和前后修改器的联动

### 和 `nx 发射器（nxEmitter）` 的联动

碰撞体看到的，是发射器已经放出来的粒子。

所以当你觉得碰撞看起来不对时，先别急着只改碰撞体。  
也要回头看：

- 粒子出生位置是不是已经偏了
- 粒子初始速度是不是太大
- 粒子是不是从错误的一侧冲向碰撞表面

这些都会直接改变你对碰撞结果的观察。

### 和 `nx 缓存（nxCache）` 的联动

把碰撞放在缓存之后的好处，是你能更稳定地比较“加碰撞前”和“加碰撞后”的变化。

如果你修改了碰撞设置，却感觉结果没有变化，也要顺手检查一件事：

- 当前是不是还在读旧缓存，或者当前主线还没有重新计算到你以为的状态

## 当前版本下最常见的几个检查点

### 粒子直接穿过去

优先按这个顺序检查：

1. 碰撞体（Colliders）列表里有没有真的加入对象
2. 对象类型是不是网格（Mesh）或曲线（Curves）
3. 这一项是不是启用状态
4. 法线方向（Normals）是不是选错了
5. 必要时先切到任意（Any）排查

### 粒子会撞，但结果很不像预期

优先检查：

- 弹力（Bounce）
- 摩擦力（Friction）
- 散射（Scatter）

这三项组合起来，决定碰撞后是更弹、更滑，还是更乱。

### 我明明加了碰撞体，为什么感觉像没有区别

除了碰撞体本身，也要顺手回头看：

- `nx 发射器（nxEmitter）` 的粒子是不是已经偏离了你以为的路径
- `nx 缓存（nxCache）` 里是不是还在回放旧结果

很多“碰撞没有变化”的现象，实际原因不只在碰撞体本身。
