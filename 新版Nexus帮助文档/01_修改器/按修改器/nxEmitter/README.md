# nx 发射器（nxEmitter）

当前文档版本：NeXus `2026.0.7092006`

`nx 发射器（nxEmitter）` 负责三件核心事情：

- 粒子从哪里出现
- 粒子什么时候出现
- 粒子出生时带着什么初始状态

## 什么时候先看它

只要你遇到下面这些现象，通常都应该先回头检查 `nx 发射器（nxEmitter）`：

- 场景里完全没有粒子
- 粒子出现了，但出生位置不对
- 粒子出现时间不对
- 粒子数量和预期差很多
- 后面的拖尾、流体、碰撞结果看起来很怪

后面的修改器经常会把问题放大，但起点多数还是发射器本身。

## 当前界面怎么理解

当前版本里，`nx 发射器（nxEmitter）` 的内容可以先按这几块来理解：

- 发射器（Emitter）：决定发射区域和发射来源
- 初始状态（Initial State）：决定粒子刚出生时的速度、半径、质量这类基础属性
- 发射（Emission）：决定发射节奏、时间和数量
- 运动继承（Motion Inheritance）：决定粒子是否继承来源对象的运动
- 显示（Display）：决定视口里怎么看粒子
- 导出（Export）：决定是否把粒子导出成 Blender 点云（Point Cloud）
- 组（Groups）：决定粒子被分到哪些组
- 修改器（Modifiers）：决定这批粒子后面要接入哪些 NeXus 修改器

## 一条最容易跑通的起步方法

1. 先创建 `nx 发射器（nxEmitter）`。
2. 在发射器（Emitter）里先选一个容易观察的形状（Shape），例如矩形（Rectangle）、圆盘（Disc）或物体（Object）。
3. 在发射（Emission）里先选一个容易理解的发射类型（Emit Type），例如单次发射（Shot）或速率（Rate）。
4. 在初始状态（Initial State）里给一个明显的速度（Speed）。
5. 打开显示（Display）里的显示粒子（Show Particles），先确认视口里真的有粒子出现。
6. 粒子出现后，再把这批粒子接到 `nx 缓存（nxCache）` 和 `nx 碰撞体（nxCollider）`。

这样做的好处是，你可以先确认“有没有粒子”，再去看“怎么缓存”和“会撞到什么”。

## 先抓住这几个最关键的控制项

### 形状（Shape）

形状（Shape）决定发射区域是什么类型。当前版本里常见的是：

- 矩形（Rectangle）
- 圆盘（Disc）
- 球体（Sphere）
- 盒体（Box）
- 物体（Object）

想先快速看到结果时，通常优先用矩形（Rectangle）或圆盘（Disc）。  
想让粒子沿模型表面、边、点或体积发射时，再切到物体（Object）。

### 发射类型（Emit Type）

发射类型（Emit Type）决定发射节奏。当前主线里最容易理解的是：

- 单次发射（Shot）：在一个时间段里打一批
- 脉冲（Pulse）：发一阵，停一阵，再继续发
- 速率（Rate）：持续产生粒子

如果你是第一次确认发射器是否正常，优先从单次发射（Shot）开始。这样更容易判断“这一批粒子到底有没有出来”。

### 单次开始（Shot Start）

单次开始（Shot Start）控制单次发射（Shot）从什么时候开始。  
当前版本代码里的默认值是 `2`。

这个值很关键，因为它会直接决定你在时间线上什么时候第一次看到粒子。  
如果你把它改得太早，部分场景里第一批粒子可能落在初始化边界附近，结果看起来像“没有产生”。

遇到这类情况时，先这样检查：

- 确认发射类型（Emit Type）确实是单次发射（Shot）
- 把单次开始（Shot Start）先放回 `2` 或更大
- 再看视口里是否开始出现粒子

### 单次持续时间（Shot Duration）

单次持续时间（Shot Duration）控制这一批发射会摊开多久。

- 值很短时，更像瞬间打出一批
- 值拉长时，这一批粒子会分散到更长时间里出生

如果你觉得一批粒子挤得太密，先拉长这里。  
如果你想做突然爆开的一团，先缩短这里。

### 脉冲开始（Pulse Start）和脉冲长度（Pulse Length）

这两个值只在脉冲（Pulse）模式下重要。

- 脉冲开始（Pulse Start）控制整套脉冲从什么时候启动
- 脉冲长度（Pulse Length）控制每次“开喷”持续多久

如果你还在确认基础发射是否正常，先不用急着进脉冲（Pulse）。  
等单次发射（Shot）或速率（Rate）已经清楚后，再回来做节奏变化会更稳。

### 速度（Speed）

速度（Speed）决定粒子刚出生时被推出去的初始速度。

如果粒子看起来像堆在原地，先提高速度（Speed）。  
如果粒子一出现就飞得过远，先降低速度（Speed）。

### 发射方向预览

`nx 发射器（nxEmitter）` 里的发射方向和旋转用于视口预览，帮助判断粒子从发射器的哪一侧、往哪个方向发出。

这组预览不会写入粒子的实际旋转。
当粒子用于实例化有前后方向的模型时，模型朝向和自转需要由 `nx 自旋（nxSpin）` 控制。

如果粒子由 `nx 流场（nxFlowField）` 改变移动方向，想让模型始终指向行进方向，可以在 `nx 自旋（nxSpin）` 添加切向（Tangential）层。
切向轴（Tangential Axis）要选模型自身用作“前方”的局部轴；模型朝向歪了时，再用旋转偏移（Rotation Offset）校正。

### 半径（Radius）和质量（Mass）

半径（Radius）和质量（Mass）都属于出生时的基础状态。

- 半径（Radius）会影响粒子大小，也会影响某些规则排布时的密度判断
- 质量（Mass）会影响粒子后面受力和碰撞时的反应

如果当前目标只是先跑通主线，通常先把速度（Speed）看顺，再回来细调半径（Radius）和质量（Mass）。

### 子帧发射（Subframe Emit）

子帧发射（Subframe Emit）决定发射是不是只落在整帧上，还是会分布到更细的子帧时间里。

如果你觉得连续喷射有断续感，或者高速运动时发射显得太跳，优先检查这里。  
想先简化判断时，可以先用更容易观察的发射方式把主线跑通，再决定是否打开子帧发射（Subframe Emit）。

## 运动继承（Motion Inheritance）什么时候要开

运动继承（Motion Inheritance）用于把来源对象本身的运动带给新出生的粒子。

它适合这些情况：

- 发射来源对象本身在移动
- 发射来源对象本身在旋转
- 你希望粒子一出生就跟着来源带出动势

如果你的发射来源是静止的，或者你还在排查粒子移动方向，可以先不启用运动继承（Motion Inheritance），先把基础发射理顺。

## 显示（Display）和导出（Export）要怎么读

显示（Display）主要解决“怎么看”，导出（Export）主要解决“怎么把结果带到 Blender 其他流程里”。

它们都很重要，但判断时要注意顺序：

- 先确认有没有粒子
- 再确认视口里怎么显示
- 最后再决定要不要导出点云（Point Cloud）

如果你只是看不到粒子，先不要急着怀疑发射逻辑坏了。先检查：

- 显示粒子（Show Particles）有没有打开
- 当前显示模式（Display Mode）是不是太不容易观察
- 视口时间线有没有走到发射开始之后

## 和 `nx 缓存（nxCache）` 的联动

`nx 发射器（nxEmitter）` 是 `nx 缓存（nxCache）` 的上游。

这意味着：

- 发射器没有产生粒子时，缓存也不会凭空记录出结果
- 你改了发射参数后，旧缓存可能还在回放旧结果
- 如果改完发射器却看不到变化，除了回头看发射器本身，也要检查缓存是不是还在播放旧数据

## 和 `nx 碰撞体（nxCollider）` 的联动

`nx 碰撞体（nxCollider）` 会改变你对发射是否正常的判断。

同样一批粒子：

- 没有碰撞时，可能会直接穿过去
- 加上碰撞后，可能会堆积、反弹、贴着表面滑动

所以当你开始接 `nx 碰撞体（nxCollider）` 之后，如果结果突然看起来完全不一样，不一定是发射器坏了，也可能只是碰撞开始真正生效了。

## 当前版本下最常见的几个检查点

### 场景里完全没有粒子

先按这个顺序检查：

1. 当前时间线是否已经走到发射开始之后
2. 发射类型（Emit Type）是不是你以为的那个模式
3. 单次开始（Shot Start）是不是设得太早或太晚
4. 速度（Speed）是不是太低，导致你误以为没有发射
5. 显示粒子（Show Particles）是不是关闭了

### 粒子出生位置不对

优先检查：

- 形状（Shape）
- 发射来源对象
- 发射方向预览相关设置

如果你已经在用物体（Object）发射，通常还要继续回头检查来源对象本身的几何和朝向。

### 后面的拖尾、流体、碰撞结果不对

先不要急着直接改后面的修改器。  
先回到 `nx 发射器（nxEmitter）` 看三件事：

- 粒子是不是在正确的位置出生
- 粒子是不是在正确的时间出生
- 粒子初始速度是不是已经偏了

很多下游问题，真正的起点都在这里。
