# nx 流体（nxFluids）

当前文档版本：NeXus `2026.0.7011503`

`nx 流体（nxFluids）` 用来让粒子按流体方式运行。  
它会决定这批粒子怎么流动、怎么被限制在容器里、怎么处理稳定性，以及后续能看到多少液体细节。

这类场景通常会用到它：

- 想让粒子形成液体流动
- 想做带边界的流体容器
- 想控制流体是更平稳、更飞溅，还是更偏颗粒感
- 想单独提高流体求解的稳定性

## 当前界面怎么理解

当前版本里，`nx 流体（nxFluids）` 可以先按两层来理解：

- 第一层是选求解器（Solver）
- 第二层是看这个求解器对应的参数分组

求解器（Solver）切换后，界面结构会一起变化：

- `PBD 求解器（PBD）`：主参数区加 `包围盒（Bounding Box）`
- `SPH 求解器（SPH）`：主参数区加 `包围盒（Bounding Box）`
- `FLIP 求解器（FLIP）`：`域（Domain）` / `求解器精度（Solver Accuracy）`，并带 `显示（Display）`
- `仿射网格粒子求解器（APIC）`：`域（Domain）` / `求解器精度（Solver Accuracy）`，并带 `显示（Display）`

所有求解器都会带 `混合器（Mixer）`。  
这里可以按发射器或组覆盖流体设置，组的覆盖优先级高于发射器。

## 先决定用哪一种求解器

### `PBD 求解器（PBD）`

`PBD 求解器（PBD）` 这一支的界面更直接，重点放在：

- `平滑半径（Smoothing Radius）`
- `子步（Substeps）`
- 密度相关控制
- `液体（Liquid）` 分组
- `包围盒（Bounding Box）`

当你想先快速建立一个被盒体限制住的流体结果，这一支会比较直接。  
它没有 `流体类型（Fluid Type）` 切换，也没有 `域（Domain）` 页签。

这一支里的功能块可以这样看：

- `平滑半径（Smoothing Radius）`：控制粒子彼此影响的范围。值更高时，液体更容易连成整体，表面变化更圆；值更低时，局部变化更明显，也更容易显得松散。
- `子步（Substeps）`：控制这一支每帧要拆成多少次中间计算。值更高时，快速流动、撞边和局部挤压通常更稳；值更低时，计算更轻，但更容易跳动、穿透或贴边不稳。
- `缓入（Ease In）`：控制流体影响进入的过渡时间。值更高时，效果进入得更慢、更柔和；值更低时，更快进入完整效果。
- `缓出（Ease Out）`：控制流体影响退出的过渡时间。值更高时，结束前衰减更慢；值更低时，结束更干脆。
- `最小密度（Min Density）`：控制密度求解的最低迭代数。值更高时，基础体积感更容易稳住；值更低时，计算更轻，但局部松散更容易出现。
- `最大密度（Max Density）`：控制密度求解允许加到多高。值更高时，强挤压区域更容易收稳；值更低时，极端挤压下更容易发软。
- `密度压缩（Density Compression）`：控制局部被压紧时允许缩到什么程度。值更高时，被挤压的区域更容易继续缩紧，不会太快弹开；值更低时，被挤压到一定程度后更早停住，更容易回弹和散开。
- `检查密度（Check Density）`：额外检查密度是否超出范围。开启后，会继续参考下面的最大密度和迭代；关闭后，这一层额外限制不会参与。
- `粘度（Viscosity）`：控制液体有多稠。值更高时，流动更慢、更黏、更不容易散；值更低时，流动更快、更容易铺开。
- `涡度（Vorticity）`：控制卷动和旋转感。值更高时，卷流和翻滚更明显；值更低时，流动更平顺。
- `吸引力（Attraction）`：控制粒子彼此收拢的趋势。值更高时，液体更容易抱团；值更低时，粒子更容易分开。
- `排斥力（Repulsion）`：控制粒子彼此推开的趋势。值更高时，更不容易挤成一团；值更低时，局部更容易压紧。
- 这三个参数可以直接这样区分：
  `密度压缩（Density Compression）` 主要看“被压住以后，还能缩多紧”。
  `吸引力（Attraction）` 主要看“周围粒子会不会主动往一起收”。
  `排斥力（Repulsion）` 主要看“粒子会不会主动把彼此顶开”。
  如果你看到的是一团东西被压在角落或窄口里，先看 `密度压缩（Density Compression）`。
  如果你看到的是粒子整体想不想抱成团，先看 `吸引力（Attraction）`。
  如果你看到的是粒子彼此挤得太死、太贴，先看 `排斥力（Repulsion）`。
- `外部压力（External Pressure）`：给液体额外的外向撑开趋势。值更高时，体积更容易被撑开；值更低时，整体更容易回缩。
- `包围盒（Bounding Box）` / `盒体墙壁（Box Walls）`：给 `PBD 求解器（PBD）` 这一支提供容器边界。盒体更大时，流体活动空间更大；盒体更小时，更容易很快撞到边界。

### `SPH 求解器（SPH）`

`SPH 求解器（SPH）` 是四种求解器里，当前唯一带 `流体类型（Fluid Type）` 的一支。

它可以切换：

- `液体（Liquid）`
- `颗粒（Granular）`

这意味着你在这一支里，可以继续往液体方向调，也可以往颗粒方向调。  
它还会额外提供：

- `自适应子步（Adaptive Substeps）`
- `阻尼（Damping）`
- `液体（Liquid）` 分组
- `颗粒（Granular）` 分组
- `包围盒（Bounding Box）`

如果你需要在同一个流体修改器里明确切到颗粒求解，这一支最值得先看。

这一支里的功能块可以这样看：

- `流体类型（Fluid Type）`：决定当前按 `液体（Liquid）` 还是 `颗粒（Granular）` 方向求解。切到 `液体（Liquid）` 时，更适合连续流动；切到 `颗粒（Granular）` 时，更适合沙粒、堆积这类结果。
- `平滑半径（Smoothing Radius）`：控制粒子影响范围。值更高时，更容易连成连续流体；值更低时，局部细节更明显，也更容易显得散。
- `子步（Substeps）`：控制这一支每帧的基础计算次数。值更高时，更稳；值更低时，更轻但更容易跳动或穿透。
- `自适应子步（Adaptive Substeps）`：让步数跟着运动强度变化。适合有快有慢的流动。
- `自适应（Adaptive）`：开启后，`最小（Min）` 和 `库朗数（CFL）` 才会一起参与；关闭后，更接近固定步数。
- `最小（Min）`：自适应回落时允许保留的最低步数。值更高时，平稳阶段也会保持更多计算；值更低时，空闲阶段更省计算。
- `库朗数（CFL）`：控制自适应子步随速度变化的积极程度。值更低时，更容易提早加步；值更高时，更倾向保留较少步数。
- `阻尼（Damping）`：压低速度波动。值更高时，晃动更少、结果更稳；值更低时，速度保留更多，运动更活。
- `缓入（Ease In）`：控制流体影响进入的过渡时间。值更高时，进入更慢；值更低时，更快进入完整效果。
- `最小密度（Min Density）`：控制最低密度求解迭代。值更高时，基础体积感更稳；值更低时，计算更轻。
- `最大密度（Max Density）`：控制高压区域允许加到多高。值更高时，更能压稳密集区域；值更低时，强挤压下更容易发软。
- `最大压缩（Max Compression）`：控制材料允许被压缩到什么程度。值更高时，更能容忍被压紧；值更低时，更早表现出抗压反应。
- `速度校正（Velocity Correction）`：用额外的速度修正来提升稳定性。开启后，结果通常更稳；关闭后，运动会更直接。
- `检查密度（Check Density）`：额外检查密度是否超出范围。开启后，会继续参考下面的最大密度和迭代；关闭后，这一层额外限制不会参与。
- `液体（Liquid）` 分组：
  `粘度（Viscosity）` 值更高时更稠、更慢；值更低时更容易流开。
  `粘度迭代次数（Viscosity Iter）` 值更高时，粘度效果更稳；值更低时，粘稠感更容易发软。
  `涡度（大）（Vorticity (Large)）` 值更高时，大尺度翻滚更明显；值更低时，大波动更弱。
  `涡度（小）（Vorticity (Small)）` 值更高时，小尺度细碎卷动更多；值更低时，表面更安静。
  `表面张力（Surface Tension）` 值更高时，更容易收拢成团；值更低时，更容易摊开。
  `内部压力（Internal Pressure）` 值更高时，内部更容易把体积往外顶；值更低时，整体更容易回缩。
  `外部压力（External Pressure）` 值更高时，外向撑开的感觉更明显；值更低时，这种撑开趋势更弱。
- `颗粒（Granular）` 分组：
  `摩擦力（Friction）` 值更高时，更不容易滑开；值更低时，更容易滑动。
  `摩擦迭代次数（Friction Iterations）` 值更高时，堆积和接触更稳；值更低时，堆积更容易发软。
  `稳定性（Stability）` 值更高时，整体更稳；值更低时，运动更活，但更容易乱跳。
  `内聚（Cohesion）` 值更高时，更容易抱团；值更低时，更像分散颗粒。
- `包围盒（Bounding Box）` / `盒体墙壁（Box Walls）`：给 `SPH 求解器（SPH）` 这一支提供容器边界。盒体更大时，活动空间更大；盒体更小时，更容易很快撞到边界。

### `FLIP 求解器（FLIP）`

`FLIP 求解器（FLIP）` 属于域驱动的流体分支。  
这一支的核心理解方式和前两支不同，重点会放在：

- `体素大小（Voxel Size）`
- `域大小（Domain Size）`
- `重定时（Retiming）`
- `FLIP 比例（FLIP Fraction）`
- `求解器精度（Solver Accuracy）`
- `粒子重播种（Particle Reseeding）`
- `库朗数（CFL）`
- `最小子步（Min Substeps）`
- `最大子步（Max Substeps）`

如果你切到这一支，重点直接看 `域（Domain）`。

这一支里的功能块可以这样看：

- `体素大小（Voxel Size）`：控制模拟网格有多细。值更小时，液体细节更多、边缘更完整，计算也更重；值更大时，计算更轻，但体积细节更少。
- `域大小（Domain Size）`：控制这块流体能活动的整体空间范围。值更大时，活动空间更大；值更小时，更容易更快撞到域边界。
- `重定时（Retiming）`：控制这一支流体推进得更快还是更慢。值更高时，整体推进更快；值更低时，更慢。
- `FLIP 比例（FLIP Fraction）`：控制 `PIC 方法（PIC）` 和 `FLIP 方法（FLIP）` 的混合比例。值更高时，更靠近 `FLIP 方法（FLIP）`，结果更活、更容易飞溅；值更低时，更靠近 `PIC 方法（PIC）`，结果更稳、更容易被平滑。
- `粘度（Viscosity）`：值更高时，液体更稠、更慢；值更低时，更容易流开。
- `涡度（Vorticity）`：值更高时，卷动和旋转感更明显；值更低时，流动更平顺。
- `表面张力（Surface Tension）`：值更高时，更容易收拢成团；值更低时，更容易铺开。
- `弱粒子排斥（Weak Particle Repulsion）`：给粒子一点额外分离力。开启后下面的 `强度（Strength）` 才会启用；`强度（Strength）` 值更高时，粒子更容易分开，值更低时，这种分离感更弱。
- `域边界墙（Domain Boundary Walls）`：决定六个方向的边界是 `开放（Open）`、`封闭（Closed）` 还是 `销毁（Kill）`。`开放（Open）` 允许流出去，`封闭（Closed）` 会撞回去，`销毁（Kill）` 会在到边界时直接移除。
- `压力求解器（Pressure Solver）`：`精度（Accuracy）` 值更高时，更愿意继续逼近目标结果；值更低时，更早停下。`迭代（Iterations）` 值更高时，高压区域更容易收稳；值更低时，更省计算。
- `粘度求解器（Viscosity Solver）`：`精度（Accuracy）` 值更高时，粘度结果更容易收稳；值更低时，更早停下。`迭代（Iterations）` 值更高时，粘度效果更稳定；值更低时，粘稠感更容易发软。只有 `粘度（Viscosity）` 大于 `0` 时，这一组才会启用。
- `粒子重播种（Particle Reseeding）`：维持每个液体体素里的粒子数量范围。开启后才会启用 `最少粒子数（Minimum Particles）` 和 `最多粒子数（Maximum Particles）`。`最少粒子数（Minimum Particles）` 值更高时，更会主动补足稀疏区域；`最多粒子数（Maximum Particles）` 值更高时，更允许局部保留更多粒子。
- `平流（Advection）`：用 `库朗数（CFL）`、`最小子步（Min Substeps）`、`最大子步（Max Substeps）` 控制这支流体的自适应时间细分。`库朗数（CFL）` 值更低时，更容易提早加步；值更高时，更倾向保留较少步数。`最小子步（Min Substeps）` 值更高时，平稳阶段也会保持更多计算；`最大子步（Max Substeps）` 值更高时，快速阶段允许加到更高步数。
- `显示（Display）`：提供域、液体体素、墙面类型、速度等视口可视化，主要用于观察当前流体内部状态。

### `仿射网格粒子求解器（APIC）`

`仿射网格粒子求解器（APIC）` 和 `FLIP 求解器（FLIP）` 一样，都是域驱动分支。  
它们共用大部分界面结构：

- `域（Domain）`
- `求解器精度（Solver Accuracy）`
- `显示（Display）`
- `混合器（Mixer）`

两者的一个直接区别是：  
`FLIP 求解器（FLIP）` 有 `FLIP 比例（FLIP Fraction）`，`仿射网格粒子求解器（APIC）` 没有这一项。

这一支里大部分功能块和 `FLIP 求解器（FLIP）` 相同：

- `域（Domain）` 页：`体素大小（Voxel Size）`、`域大小（Domain Size）`、`重定时（Retiming）`、`粘度（Viscosity）`、`涡度（Vorticity）`、`表面张力（Surface Tension）`、`弱粒子排斥（Weak Particle Repulsion）`、`域边界墙（Domain Boundary Walls）`
  这些参数的增减表现，可以直接按上面 `FLIP 求解器（FLIP）` 一节理解。
- `求解器精度（Solver Accuracy）` 页：`压力求解器（Pressure Solver）`、`粘度求解器（Viscosity Solver）`、`粒子重播种（Particle Reseeding）`、`平流（Advection）`
  这些参数的增减表现，也和上面 `FLIP 求解器（FLIP）` 一节相同。
- `显示（Display）`：用于观察域、速度和体素可视化结果

当前最直接的界面差异就是没有 `FLIP 比例（FLIP Fraction）`。

## 最容易混淆的一件事：`包围盒（Bounding Box）` 和 `域（Domain）`

这两个概念不要混在一起看。

### `包围盒（Bounding Box）` 是给 `PBD 求解器（PBD）` 和 `SPH 求解器（SPH）` 用的

`包围盒（Bounding Box）` 只出现在 `PBD 求解器（PBD）` 和 `SPH 求解器（SPH）`。  
当前版本插件里，它默认关闭。

开启后，你会继续看到：

- `盒体大小（Box Size）`
- 六个方向的墙面类型

它的作用很直接：  
给这两类流体一个盒体容器，防止流体直接散掉或掉走。

### `域（Domain）` 是给 `FLIP 求解器（FLIP）` 和 `仿射网格粒子求解器（APIC）` 用的

`域（Domain）` 只在 `FLIP 求解器（FLIP）` 和 `仿射网格粒子求解器（APIC）` 里出现。  
这一支主要靠 `体素大小（Voxel Size）` 和 `域大小（Domain Size）` 来决定模拟空间和细节密度。

如果你已经切到这两支，还继续去找 `包围盒（Bounding Box）`，当前界面里本来就不会有。

## 时间细分要分三层来看

`nx 流体（nxFluids）` 很容易和别处的子步设置一起影响结果。  
整理时最值得先分清这三层：

- `文档（Document）` 里的 `子步（Substeps）`
- `PBD 求解器（PBD）` / `SPH 求解器（SPH）` 里的 `子步（Substeps）`
- `FLIP 求解器（FLIP）` / `仿射网格粒子求解器（APIC）` 里的 `库朗数（CFL）`、`最小子步（Min Substeps）`、`最大子步（Max Substeps）`

### `文档（Document）` 里的 `子步（Substeps）`

这一级影响的是当前场景这套 NeXus 模拟每一帧怎么细分。  
它属于更上层的时间细分。

如果这里过低，下面各求解器自己的细分设置再高，结果也可能还是不稳。  
相关说明可以直接一起看 [文档（Document）- 子步（Substeps）](C:/Users/bbs99/Desktop/chinese_help_docs_source/新版Nexus帮助文档/02_面板与功能/对象与修改器界面/文档（Document）- 子步（Substeps）.md)。

### `PBD 求解器（PBD）` 和 `SPH 求解器（SPH）` 里的 `子步（Substeps）`

这两支都有直接的 `子步（Substeps）` 数值。  
当前版本插件里，两者默认值都是 `6`。

它控制的是这一块流体在每一帧里要拆成多少次中间计算。

- 提高后，快速流动、紧密接触、撞击边界时通常更稳
- 降低后，计算更轻，但更容易出现跳动、穿透、贴边不稳

如果你看到的是流体整体不稳、碰边抖动、粒子一下子穿出去，这一层通常值得先看。

### `SPH 求解器（SPH）` 里的 `自适应子步（Adaptive Substeps）`

`SPH 求解器（SPH）` 的 `子步（Substeps）` 还可以展开出一组自适应设置。  
当前版本插件里，这组展开面板默认收起，里面的 `自适应（Adaptive）` 默认开启。

这一组的理解方式可以先抓三点：

- `子步（Substeps）`：允许提高到的主要步数
- `最小子步（Min Substeps）`：自适应回落时允许保留的最低步数
- `库朗数（CFL）`：控制自适应子步如何随着运动强度变化

如果场景里有的时段很平稳，有的时段突然很快，这组设置会比固定步数更值得看。

### `FLIP 求解器（FLIP）` 和 `仿射网格粒子求解器（APIC）` 里的 `库朗数（CFL）`

这两支没有直接叫 `子步（Substeps）` 的主滑块。  
当前版本插件把这一层放在 `平流（Advection）` 分组里：

- `库朗数（CFL）`
- `最小子步（Min Substeps）`
- `最大子步（Max Substeps）`

当前版本插件里，默认值分别是：

- `库朗数（CFL）`：`0.7`
- `最小子步（Min Substeps）`：`1`
- `最大子步（Max Substeps）`：`3`

可以先这样理解：

- `库朗数（CFL）` 决定自适应时间细分有多积极
- `最小子步（Min Substeps）` 和 `最大子步（Max Substeps）` 一起限定允许变化的区间

如果你把最小子步（Min Substeps）和最大子步（Max Substeps）设成同一个值，这一层就会更接近固定步数。

## 当前界面里几块最值得先看的区域

### `SPH 求解器（SPH）` 的 `流体类型（Fluid Type）`

当你切到 `SPH 求解器（SPH）`，先确认自己现在是在：

- `液体（Liquid）`
- `颗粒（Granular）`

因为这会直接改变你后面该重点看的参数组。  
液体方向更常回头看：

- `粘度（Viscosity）`
- `涡度（Vorticity）`
- `表面张力（Surface Tension）`
- `内部压力（Internal Pressure）`
- `外部压力（External Pressure）`

颗粒方向更常回头看：

- `摩擦力（Friction）`
- `摩擦迭代次数（Friction Iterations）`
- `稳定性（Stability）`
- `内聚（Cohesion）`

### `FLIP 求解器（FLIP）` 的 `FLIP 比例（FLIP Fraction）`

`FLIP 比例（FLIP Fraction）` 只在 `FLIP 求解器（FLIP）` 里出现。  
当前版本插件默认值是 `95%`。

它用于在 `PIC 方法（PIC）` 和 `FLIP 方法（FLIP）` 之间混合。

- 往更高的 `FLIP 比例（FLIP Fraction）` 调时，结果会更靠近 `FLIP 方法（FLIP）` 一侧
- 往更低的 `FLIP 比例（FLIP Fraction）` 调时，结果会更靠近 `PIC 方法（PIC）` 一侧

如果你当前在 `仿射网格粒子求解器（APIC）` 里找这项，当前界面本来就不会出现。

### `FLIP 求解器（FLIP）` / `仿射网格粒子求解器（APIC）` 的 `粒子重播种（Particle Reseeding）`

`粒子重播种（Particle Reseeding）` 位于 `求解器精度（Solver Accuracy）` 页里。  
当前版本插件里，它默认关闭。

开启后，会继续用到：

- `最少粒子数（Minimum Particles）`
- `最多粒子数（Maximum Particles）`

这组设置的重点是维持每个液体体素里的粒子数量范围。  
当结果出现局部太稀、细节断掉、流体分布不均时，这里会更值得检查。

### `FLIP 求解器（FLIP）` / `仿射网格粒子求解器（APIC）` 的 `弱粒子排斥（Weak Particle Repulsion）`

这项位于 `域（Domain）` 页里。  
当前版本插件里，它默认关闭。

开启后，下面的强度参数才会启用。  
如果你打开开关后，下面的强度还是灰的，先回头确认这里还没启用。

## 边界墙怎么理解

`包围盒（Bounding Box）` 和 `域边界墙（Domain Boundary Walls）` 都会用到同一套边界类型：

- `开放（Open）`：粒子可以穿过去
- `封闭（Closed）`：粒子会撞上去
- `销毁（Kill）`：粒子碰到这里会被移除

这类设置最容易影响的是：

- 流体会不会被关在容器里
- 流体能不能从某个方向流出去
- 边界附近为什么突然少了一批粒子

如果你看到流体一到边缘就不见了，先检查该方向是否设成了 `销毁（Kill）`。

## 和其他常见修改器一起看时，最容易卡住的地方

### 和 `nx 发射器（nxEmitter）`

流体的起点仍然来自发射器。  
发射数量、半径、初速度、发射节奏，都会直接改变流体一开始的密度和运动强度。

如果流体一出生就挤爆、飞散、太稀，先回头看 `nx 发射器（nxEmitter）`，不要只盯着流体求解器。

### 和 `nx 碰撞体（nxCollider）`

如果场景里有容器、地面、导流物体，结果会同时受到流体求解和碰撞体设置影响。  
贴边抖动、穿透、弹开或卡边，这类现象经常同时和流体求解、碰撞体设置有关。

当你已经提高了流体自己的时间细分，结果还是穿透或贴边不稳，也要一起检查 `nx 碰撞体（nxCollider）`。

### 和 `nx 粒子-粒子碰撞（nxPPCollisions）`

当场景里同时用了 `nx 粒子-粒子碰撞（nxPPCollisions）`，粒子之间的接触稳定性会更明显影响流体观感。  
这类场景里，`文档（Document）` 的 `子步（Substeps）`、流体自己的时间细分、粒子-粒子碰撞自己的时间细分，往往需要一起判断。

### 和 `nx 缓存（nxCache）`

如果你已经改了 `nx 流体（nxFluids）` 的参数，结果却完全没变化，别忘了回头检查缓存状态。  
缓存内容没有更新时，很容易误以为某个参数没生效。

## 当前版本下最常见的检查顺序

### 流体直接散掉或掉走

优先检查：

1. 先确认当前是否是 `PBD 求解器（PBD）` 或 `SPH 求解器（SPH）`
2. `包围盒（Bounding Box）` 是否已经开启
3. `盒体大小（Box Size）` 是否足够包住流体
4. 边界墙是否被设成了 `开放（Open）`

### 流体在边界附近抖动、穿透或突然跳动

优先检查：

1. `文档（Document）` 的 `子步（Substeps）`
2. 当前求解器自己的时间细分设置
3. `nx 碰撞体（nxCollider）`
4. 当前流动速度是否已经高到超出现在这套细分范围

### 切到 `FLIP 求解器（FLIP）` 或 `仿射网格粒子求解器（APIC）` 后找不到盒体

这是当前版本里的正常界面表现。  
这两支看的是 `域（Domain）`，不再使用 `包围盒（Bounding Box）`。

### 改了参数却看不到变化

优先检查：

1. 先确认当前缓存是否还是旧结果
2. 先确认你改的是当前求解器实际在用的那一组参数
3. 先确认这个参数是否需要先启用上层开关才会生效

例如：

- `粒子重播种（Particle Reseeding）` 需要先启用
- `弱粒子排斥（Weak Particle Repulsion）` 需要先启用
- `粘度求解器（Viscosity Solver）` 相关项只有在 `粘度（Viscosity）` 大于 `0` 时才会启用
