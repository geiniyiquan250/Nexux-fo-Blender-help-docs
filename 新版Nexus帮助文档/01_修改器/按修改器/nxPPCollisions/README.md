# nx 粒子-粒子碰撞（nxPPCollisions）

当前文档版本：NeXus `2026.0.7011503`

`nx 粒子-粒子碰撞（nxPPCollisions）` 用来处理粒子和粒子之间的碰撞。  
它适合解决这类结果：

- 粒子应该互相挤开，却直接穿在一起
- 一堆粒子落下来时，堆积感不明显
- 粒子彼此接触后太滑、太弹，或者乱跳
- 高速运动时，粒子之间的接触不稳定

## 当前界面怎么理解

当前版本里，`nx 粒子-粒子碰撞（nxPPCollisions）` 的参数可以先分成三块来理解：

- 开关范围：自碰撞（Self Collisions）
- 碰撞后的手感：弹力（Bounce）、变化（Variation）、摩擦力（Friction）、散射（Scatter）、内聚（Cohesion）
- 稳定性和精度：子步（Substeps）、迭代（Iterations）

如果你第一次接触它，最值得先看的通常只有这几项：

- 自碰撞（Self Collisions）
- 弹力（Bounce）
- 摩擦力（Friction）
- 子步（Substeps）
- 迭代（Iterations）

## 它解决的是什么问题

这个修改器关心的是粒子彼此之间会不会撞开、撞开后像什么、以及密集接触时稳不稳。

把它加进来之后，最直接的变化通常是：

- 粒子更容易分开，不再轻易互相穿过去
- 粒子堆在一起时，更容易形成堆积、挤压和接触层次
- 碰撞后的反应可以更弹、更滑、更散，或者更黏

如果你的场景重点是“粒子要互相影响”，这就是很常用的一步。  
如果你要解决的是“粒子撞场景物体”，那一类问题更应该先看 `nx 碰撞体（nxCollider）`。

## 一条最容易跑通的起步方法

1. 先确认 `nx 发射器（nxEmitter）` 已经能稳定产生足够多的粒子。
2. 先让粒子进入一个容易看到堆积或互相接触的场景。
3. 添加 `nx 粒子-粒子碰撞（nxPPCollisions）`。
4. 先保持自碰撞（Self Collisions）开启。
5. 先只观察粒子有没有开始彼此挤开。
6. 再根据结果去调弹力（Bounce）、摩擦力（Friction）、子步（Substeps）和迭代（Iterations）。

这样做的好处是，你能先确认“有没有开始互相碰撞”，再去判断“碰撞后的手感对不对”。

## 先抓住这几个最关键的控制项

### 自碰撞（Self Collisions）

自碰撞（Self Collisions）控制同一发射器出来的粒子，是否彼此发生碰撞。

- 开启后，同一批粒子会开始互相挤开
- 关闭后，同一批粒子更容易彼此穿过

当前版本默认是开启。  
如果你明明加了这个修改器，结果却还是像一股粒子直接穿在一起，先回头检查这里有没有被关掉。

### 弹力（Bounce）

弹力（Bounce）控制粒子碰撞后保留多少反弹趋势。

- 值越高，碰撞后越容易弹开
- 值越低，碰撞后越像把速度吃掉一部分

当前版本默认值是 `1.0`。  
这个起点会让碰撞反应比较明显，方便你先确认它已经开始生效。

如果你想做更硬、更跳的颗粒感，可以先提高弹力（Bounce）。  
如果你想让结果更沉、更稳、更像堆积，通常要先把弹力（Bounce）往下压。

### 变化（Variation）

变化（Variation）会给每次碰撞加上随机减弱。

这里有两处变化（Variation）：

- 弹力（Bounce）下面的变化（Variation）
- 摩擦力（Friction）下面的变化（Variation）

它们的共同作用是让每次碰撞别那么整齐。

- 值越高，单次碰撞结果越不统一
- 值越低，碰撞反应越稳定、越可预测

如果你还在排查基础碰撞是否正常，通常先把变化（Variation）保持低一些。  
等主体行为已经对了，再回来加随机差异会更容易判断。

### 摩擦力（Friction）

摩擦力（Friction）控制粒子接触后沿彼此表面滑动时的阻力。

- 值越高，粒子越不容易彼此滑开
- 值越低，粒子越容易继续滑动

当前版本默认值是 `0.0`。  
这个起点更容易先看到“碰撞有没有发生”，不会一上来就把所有接触都拉得很黏。

如果你的结果太滑、堆不住、轻轻一碰就散开，通常可以先提高摩擦力（Friction）。  
如果你想保留更流畅的滑动感，就保持低一些。

### 散射（Scatter）

散射（Scatter）控制碰撞后方向被打散的程度。

- 值越高，碰撞后的方向越乱
- 值越低，碰撞后的轨迹越稳定

当前版本默认值是 `0.0`。  
如果你只是想先做稳定可控的粒子接触，通常先不要急着把散射（Scatter）拉高。

### 内聚（Cohesion）

内聚（Cohesion）控制附近粒子之间的吸附倾向。

- 值越高，粒子越容易黏成团
- 值越低，粒子更像单独分开的颗粒

当前版本默认值是 `0.0`。  
它更适合做湿沙、潮湿颗粒、抱团这类效果。

如果你只是想先让粒子正常互相挤开，通常先把内聚（Cohesion）保持低值。  
等基础碰撞已经稳定，再回来加内聚（Cohesion），更容易看出它带来的变化。

## 稳定性最关键的两项

### 子步（Substeps）

子步（Substeps）控制这一块粒子-粒子碰撞在每一帧里会分成多少次中间计算。

- 值越高，接触通常更硬、更稳，也更不容易漏碰撞
- 值越低，计算更轻，但高速接触更容易变软、穿透或不稳

当前版本默认值是 `10`。  
这个默认值已经比很多基础参数更偏向稳定性，因为粒子彼此碰撞本来就很容易在高速或密集接触时出问题。

如果你看到下面这些现象，通常先检查子步（Substeps）：

- 粒子彼此穿过去
- 堆积时一层层压进去
- 高速接触时接触感太软

### 迭代（Iterations）

迭代（Iterations）控制每个子步（Substeps）里，求解器会重复处理多少轮接触。

- 值越高，深层堆积和密集接触通常更稳
- 值越低，计算更轻，但堆得高、压得深时更容易松、软或互相压进去了

当前版本默认值是 `2`。  
当前版本里，`2` 到 `4` 更适合密集堆积场景。

如果你的问题主要出现在：

- 粒子堆得很高
- 一大团粒子互相挤压
- 表面看起来碰上了，但内部还在互相压穿

通常可以先提高迭代（Iterations）。

## 子步（Substeps）和迭代（Iterations）怎么一起看

这两个值不要拆开看。

可以先这样理解：

- 子步（Substeps）更偏向把一帧里的碰撞过程切得更细
- 迭代（Iterations）更偏向在每一次细分之后，把接触再多压几轮

常见判断顺序可以这样用：

1. 高速运动漏碰撞，先看子步（Substeps）
2. 堆积太软、深层接触不稳，先看迭代（Iterations）
3. 两类问题同时存在时，通常先加子步（Substeps），再补迭代（Iterations）

## 和其他常见修改器的联动

### 和 `文档（Document）- 子步（Substeps）` 的联动

`文档（Document）` 里的子步（Substeps），影响的是当前场景这套 NeXus 模拟每一帧怎么细分。  
`nx 粒子-粒子碰撞（nxPPCollisions）` 里的子步（Substeps），影响的是这一块粒子-粒子碰撞自己怎么细分。

如果你已经提高了这里的子步（Substeps），结果还是不稳，也要回头一起检查 `文档（Document）` 里的子步（Substeps）。

### 和 `nx 发射器（nxEmitter）` 的联动

粒子之间会不会很快撞成一团，和发射器本身关系很大。

这些上游条件都会明显改变结果：

- 粒子出生数量
- 粒子出生半径
- 粒子初始速度
- 发射节奏是不是太密

如果粒子一出生就挤爆、互相压穿，别只盯着碰撞参数。  
也要回头检查发射器是不是把粒子塞得太密、推得太快。

### 和 `nx 碰撞体（nxCollider）` 的联动

`nx 粒子-粒子碰撞（nxPPCollisions）` 负责粒子和粒子。  
`nx 碰撞体（nxCollider）` 负责粒子和场景对象。

实际场景里，这两个经常一起用。  
你看到的结果，常常会同时受到“粒子彼此怎么撞”和“粒子撞墙、撞地面后怎么反应”的共同影响。

### 和 `nx 流体（nxFluids）` 的联动

如果你场景里还用了 `nx 流体（nxFluids）`，结果会更容易受到时间细分和接触稳定性的影响。  
这类情况下，`文档（Document）` 的子步（Substeps）和这里的子步（Substeps）都更值得一起看。

## 当前版本下最常见的几个检查点

### 粒子还是互相穿过去

优先按这个顺序检查：

1. 自碰撞（Self Collisions）是不是开着
2. 粒子是不是来自同一发射器
3. 子步（Substeps）是不是太低
4. 粒子出生速度是不是太快
5. 粒子出生得是不是太密

### 粒子会撞，但像一堆弹珠乱跳

优先检查：

- 弹力（Bounce）是不是太高
- 散射（Scatter）是不是太高
- 摩擦力（Friction）是不是太低

### 粒子会接触，但堆不住、站不稳

优先检查：

- 摩擦力（Friction）
- 子步（Substeps）
- 迭代（Iterations）

### 结果太黏，整团抱死

优先检查：

- 内聚（Cohesion）是不是太高
- 摩擦力（Friction）是不是太高

## 什么时候先不要急着改它

如果你现在看到的问题更像下面这些，通常先别把主要精力放在 `nx 粒子-粒子碰撞（nxPPCollisions）`：

- 粒子根本没有发射出来
- 粒子撞场景物体时有问题
- 缓存结果和当前修改不一致

这几类情况通常更应该先回头看：

- `nx 发射器（nxEmitter）`
- `nx 碰撞体（nxCollider）`
- `nx 缓存（nxCache）`
